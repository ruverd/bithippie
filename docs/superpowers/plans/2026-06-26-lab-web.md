# Lab Experiment Tracking — Frontend (Plan 3 of 3) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the *structure* of the React frontend in `apps/web` — the OpenAPI→Kubb typed API layer, dev connectivity, a data-wired route skeleton, and the record-measurement form primitive — leaving page layouts/visual design to the user to fill in page by page.

**Architecture:** The API (Plan 2) exposes OpenAPI at `/openapi/json`. An `apps/api` script dumps that document to `apps/api/openapi.json`; Kubb generates TypeScript types, a fetch client, TanStack Query hooks, and Zod schemas (grouped by tag) into `apps/web/src/generated`. Pages consume the generated hooks directly under an existing `QueryClientProvider` + React Router. In dev, Vite proxies `/api` → the API on `:3000` (no CORS). The one write flow (`POST /experiments/:id/measurements`) gets a real RHF + Zod form built on a `valueType`-driven `MeasurementValueField`.

**Tech Stack:** Bun, React 19, Vite, TypeScript, `react-router-dom` v7, `@tanstack/react-query` v5, `react-hook-form` + `@hookform/resolvers` + Zod, Kubb v4 (`@kubb/plugin-oas|ts|client|react-query|zod`), `@lab/shared`, Vitest, Storybook, Playwright.

## Global Constraints

- **Extend the existing `apps/web` scaffold — do not recreate it.** Reuse the already-wired `QueryClientProvider` (`src/app/providers.tsx`), `createBrowserRouter` (`src/app/router.tsx`), `Layout` (`src/app/layout.tsx`), Vite alias `@ → src`, and `src/test/setup.ts`.
- **`src/generated/` is generated, never hand-edited.** It stays git-tracked (so the app type-checks without a codegen run) but must be fully reproducible by `kubb generate`.
- **No new runtime dependencies** beyond what `apps/web/package.json` already declares. Transport is native **fetch** (Kubb's default client). Kubb plugins are already in `devDependencies`.
- **Backend is the source of truth** for all validation. The frontend mirrors rules for UX only, via `@lab/shared` (exports `MeasurementValueType`, `validateMeasurementValue`, `measurementValueInputSchema`) and generated Zod schemas.
- **API base URL** comes from `import.meta.env.VITE_API_URL`, defaulting to `/api` (same-origin, proxied in dev). The API runs on **port 3000**.
- **The user owns page layouts and visual design.** This plan builds *data-wired placeholder pages* (call the hook, handle loading/error/empty, dump raw data) and functional-but-minimally-styled form structure. Do not invest in visual design.
- **Kubb v4 / generated-name verification:** Kubb derives exact hook/type/schema export names from the OpenAPI `operationId`/path/tag and the installed `@kubb/*@4.39` API. Where this plan shows a generated identifier (e.g. `useGetProjects`), treat it as the *expected* name — after Task 1 runs codegen, open `src/generated/` and use the **actual** exported names. This is the one place names are discovered, not invented.

import { writeFileSync } from "node:fs";
import { buildApp } from "../app";
import { buildContainer } from "../container";
import { getPrisma } from "../infrastructure/prisma";

// Build the app and ask it for its own OpenAPI document. No DB connection is
// made: the /openapi/json handler touches no repository, and PrismaClient
// connects lazily (only on the first query).
const app = buildApp(buildContainer(getPrisma()));
const res = await app.handle(new Request("http://localhost/openapi/json"));
if (res.status !== 200) {
  throw new Error(`OpenAPI export failed: status ${res.status}`);
}
const doc = await res.json();
const outUrl = new URL("../../openapi.json", import.meta.url); // -> apps/api/openapi.json
writeFileSync(outUrl, `${JSON.stringify(doc, null, 2)}\n`);
// eslint-disable-next-line no-console
console.log(`Wrote ${outUrl.pathname}`);
```

import { pluginOas } from "@kubb/plugin-oas";
import { pluginTs } from "@kubb/plugin-ts";
import { pluginZod } from "@kubb/plugin-zod";
import { pluginClient } from "@kubb/plugin-client";
import { pluginReactQuery } from "@kubb/plugin-react-query";

export default defineConfig({
  root: ".",
  input: { path: "../api/openapi.json" },
  output: { path: "./src/generated", clean: true },
  plugins: [
    pluginOas(),
    pluginTs({ output: { path: "types" }, group: { type: "tag" } }),
    pluginZod({ output: { path: "zod" }, group: { type: "tag" } }),
    pluginClient({
      output: { path: "clients" },
      group: { type: "tag" },
      client: "fetch", // Kubb's bundled fetch client (no axios dependency)
    }),
    pluginReactQuery({
      output: { path: "hooks" },
      group: { type: "tag" },
      client: "fetch",
      parser: "zod", // validate responses with the generated Zod schemas
    }),
  ],
});
```

> If a v4 option key differs (e.g. `client` expects `{ importPath: "@kubb/plugin-client/clients/fetch" }`, or grouping is `group: { type: "tag", name: ... }`), adjust to the installed API — do not change the plugin set or drop fetch/zod/by-tag. Note any deviation in your report.

- [ ] **Step 4: Implement `MeasurementValueField`** — `apps/web/src/features/measurements/MeasurementValueField.tsx`

Native form elements (Tailwind classes only) so the task is self-contained — no shadcn registry fetch. The user may later swap these for styled `components/ui` primitives.

```tsx
import type { MeasurementValueType } from "@lab/shared";

interface Props {
  valueType: MeasurementValueType;
  value: string;
  onChange: (value: string) => void;
  allowedCategories?: string[];
  unit?: string;
  onUnitChange?: (unit: string) => void;
}

export function MeasurementValueField({
  valueType,
  value,
  onChange,
  allowedCategories = [],
  unit,
  onUnitChange,
}: Props) {
  if (valueType === "NUMERIC") {
    return (
      <div className="flex gap-2">
        <input
          type="number"
          inputMode="decimal"
          value={value}
          onChange={(e) => onChange(e.target.value)}
          className="rounded border px-2 py-1"
        />
        <input
          type="text"
          aria-label="unit"
          placeholder="unit"
          value={unit ?? ""}
          onChange={(e) => onUnitChange?.(e.target.value)}
          className="w-24 rounded border px-2 py-1"
        />
      </div>
    );
  }
  if (valueType === "CATEGORICAL") {
    return (
      <select
        value={value}
        onChange={(e) => onChange(e.target.value)}
        className="rounded border px-2 py-1"
      >
        <option value="" disabled>
          Select…
        </option>
        {allowedCategories.map((c) => (
          <option key={c} value={c}>
            {c}
          </option>
        ))}
      </select>
    );
  }
  return (
    <textarea
      value={value}
      onChange={(e) => onChange(e.target.value)}
      className="rounded border px-2 py-1"
    />
  );
}
```

Wire RHF + Zod resolver + the generated hooks + `MeasurementValueField`. Replace the generated hook imports/names with the actual ones; if the generated body Zod schema is convenient, prefer it over `@lab/shared`'s `measurementValueInputSchema` (both describe the same body).

```tsx
import { zodResolver } from "@hookform/resolvers/zod";
import { measurementValueInputSchema } from "@lab/shared";
import { useQueryClient } from "@tanstack/react-query";
import { Controller, useForm } from "react-hook-form";
import type { z } from "zod";
import { MeasurementValueField } from "./MeasurementValueField";
// Adjust these to the actual generated names/paths from Task 1:
import { useGetMeasurementDefinitions } from "@/generated/hooks/measurementDefinitions/useGetMeasurementDefinitions";
import { useCreateMeasurement } from "@/generated/hooks/measurements/useCreateMeasurement";

type FormValues = z.infer<typeof measurementValueInputSchema>;

export function CreateMeasurementForm({ experimentId }: { experimentId: string }) {
  const queryClient = useQueryClient();
  const definitions = useGetMeasurementDefinitions();
  const create = useCreateMeasurement();
  const { control, handleSubmit, watch, register } = useForm<FormValues>({
    resolver: zodResolver(measurementValueInputSchema),
    defaultValues: { measurementDefinitionId: "" },
  });

  const selectedId = watch("measurementDefinitionId");
  const def = (definitions.data ?? []).find((d) => d.id === selectedId);

  const onSubmit = handleSubmit((values) => {
    create.mutate(
      { experimentId, data: values }, // adjust arg shape to the generated mutation signature
      { onSuccess: () => queryClient.invalidateQueries({ queryKey: ["experiments", experimentId, "measurements"] }) },
    );
  });

  return (
    <form onSubmit={onSubmit} className="flex flex-col gap-3">
      <label className="flex flex-col gap-1">
        <span>Definition</span>
        <select aria-label="definition" {...register("measurementDefinitionId")} className="rounded border px-2 py-1">
          <option value="" disabled>
            Select…
          </option>
          {(definitions.data ?? []).map((d) => (
            <option key={d.id} value={d.id}>
              {d.name}
            </option>
          ))}
        </select>
      </label>

      {def && (
        <Controller
          control={control}
          name={def.valueType === "NUMERIC" ? "numericValue" : def.valueType === "CATEGORICAL" ? "categoricalValue" : "textValue"}
          render={({ field }) => (
            <label className="flex flex-col gap-1">
              <span>Value</span>
              <MeasurementValueField
                valueType={def.valueType}
                allowedCategories={def.allowedCategories}
                value={field.value == null ? "" : String(field.value)}
                onChange={(v) => field.onChange(def.valueType === "NUMERIC" ? Number(v) : v)}
              />
            </label>
          )}
        />
      )}

      <button type="submit" disabled={create.isPending} className="self-start rounded border px-3 py-1">
        Save
      </button>
    </form>
  );
}
```

> The `combobox` accessible name for the value select comes from the `<label><span>Value</span>` wrapper; the test queries `name: /value/i`. The mutation arg shape (`{ experimentId, data }`) must match the generated mutation's actual signature — adjust per `src/generated`.
